### Summary

This was the inaugural kickoff conversation for jj-dev — Jens Johnson's personal portfolio website. ### What I Learned

- **Aesthetic north star**: earthy, nature, modern, technological, Cali laid-back. This needs to be translated into concrete design tokens (color, type, spacing) in Sketch before any CSS is written.
- **Tech stack is fairly locked**: Nuxt + TypeScript + Tailwind is the clear preference, consistent with all reference projects. MongoDB for DB. The real open questions are hosting, monorepo structure, and content management strategy.
- **Reference project patterns**: All four reference projects (LatticeCraft, FWB API, PES, CPTL) use Nx, Nuxt 4, ESLint 10 + @stylistic, commitlint + Commitizen, direnv + NVM, and GitHub Actions. These are the established defaults.
- **Integrations available**: Todoist (Kanban tracking), Sketch MCP (design), GitHub (full access including CI/CD).
- **The most critical Day 1 decision is content architecture** — it's the one that cascades into everything else (hosting, rendering strategy, auth, admin needs, API design).

### Key Recommendations Made

1. **Nuxt Content** for editorial content (blog, projects, recipes) — Git-native, no CMS, no auth needed
2. **Vercel** for hosting — zero-config Nuxt support, preview deploys per PR, free tier
3. **Single Nuxt app** (skip Nx for now) — personal site doesn't warrant monorepo overhead yet
4. **Tailwind + Nuxt UI** — Tailwind-only for layout, Nuxt UI for complex components
5. **Design first** — define tokens in Sketch before touching `tailwind.config.ts`
